# ClaudeTower (가제)

[한국어](./README.md) | [English](./README.en.md)

Claude Code용 상태표시줄 + (예정) 계정 자동전환 CLI 도구.

> **지금 상태(중요)**: 이 프로젝트는 **Phase 1**까지만 구현되어 있습니다. 아래 "① 상태표시줄"은 지금 실제로 동작합니다. "② 계정 자동전환"은 **코드 자체가 아직 존재하지 않습니다**(꺼져있는 게 아니라 안 만들어졌습니다) — Phase 2에서 사용자 동의 절차와 함께 추가될 예정입니다.

---

## ① 상태표시줄 (항상 안전, 설치만 하면 끝)

Claude Code 화면 아래에 지금 작업 중인 프로젝트 위치·컨텍스트 사용량·비용·사용률을 보여줍니다. **여러분의 Claude 계정 정보나 비밀번호를 전혀 다루지 않습니다.**

### 사전 준비물

- [Node.js](https://nodejs.org) 22 이상 — **지금은 개발 중이라 필요하지만, 최종 배포판(SEA 바이너리)은 Node.js 설치 없이도 실행되도록 설계되어 있습니다.**
- Claude Code 최신 버전
- Git (저장소를 직접 받아 빌드하는 경우에만)

### 빠른 시작 (5단계)

1. 저장소 받기
   ```bash
   git clone https://github.com/sodam-ai/ClaudeTower.git
   cd ClaudeTower
   ```
2. 개발 도구 설치
   ```bash
   npm install
   ```
3. 배포용 실행파일 만들기
   ```bash
   npm run build
   ```
   `dist/` 폴더에 여러분 운영체제에 맞는 실행파일이 생깁니다(예: `claudetower-win-x64.exe`).
4. 상태표시줄 등록하기
   ```bash
   node bin/claudetower.js setup
   ```
   화면에 뜨는 질문(위치/컨텍스트/비용/사용률 각각 표시할지)에 Y 또는 N으로 답하면 끝입니다.
5. Claude Code에서 다음 대화부터 상태표시줄이 바로 보입니다(재시작 불필요).

> 아직 공식 배포(`curl`/`PowerShell` 한 줄 설치, `npm install -g`)는 준비 중입니다 — 자세한 사유는 아래 "설치 방법" 참고.

### 설치 방법 (배포 채널별 현재 상태)

| 방법 | 상태 | 비고 |
|---|---|---|
| 소스에서 직접 빌드 (위 5단계) | ✅ 지금 가능 | 개발자용, Node.js 필요 |
| `curl`/`PowerShell` 원라이너 (`install.sh`/`install.ps1`) | ⏳ 스크립트는 완성, **GitHub Release 발행 대기 중** | Node.js 불필요, 완성되면 가장 쉬운 방법 |
| `npm install -g` | ❌ 알려진 문제로 현재 비공개 저장소에서 동작 안 함 | 저장소가 공개 전환되면 재검증 예정 |

### 명령어 목록 (지금 실제로 있는 것만)

- `claudetower --version` / `--help`
- `claudetower setup` — 상태표시줄 항목 선택 + Claude Code 설정 자동 등록
- `claudetower statusline` — Claude Code가 내부적으로 호출하는 렌더러(직접 실행할 일 없음)

> `accounts`, `config` 등 계정 관련 명령은 **아직 존재하지 않습니다**(Phase 2 예정).

### 보안·데이터 흐름

- 아무것도 외부로 전송하지 않습니다. 여러분 컴퓨터 안에서만 동작합니다.
- Claude Code가 매번 상태표시줄 프로그램에 현재 상황(프로젝트 경로, 컨텍스트 사용량 등)을 전달하면, 그 값을 화면에 예쁘게 보여주기만 합니다 — 저장하지 않습니다.

### 아키텍처 (비유로 설명)

이 프로그램은 두 개의 완전히 분리된 방으로 나뉘어 있습니다. "상태표시줄 방"은 화면에 정보를 보여주기만 하는 방이라 늘 안전합니다. "계정전환 방"은 아직 지어지지도 않았습니다 — 나중에 지어지더라도 상태표시줄 방과는 열쇠가 통하지 않는 별도의 방입니다. 코드 검사 도구(ESLint)와 자동 테스트가 매번 두 방 사이에 몰래 문이 생기지 않았는지 확인합니다.

**이 기능은 코드 자체가 아직 존재하지 않습니다.** 완성되면 "여러분의 Claude 계정이 정지될 수도 있는 위험이 있습니다. 켜지 않아도 상태표시줄은 정상 작동합니다"라는 경고와 함께 명시적 동의 절차를 거쳐야만 켤 수 있도록 설계되어 있습니다(자세한 설계는 [`.PRD/01_PRD.md`](./.PRD/01_PRD.md) 참고).

## 트러블슈팅

- **상태표시줄이 안 보여요**: Claude Code와 새로 상호작용을 한 번 해보세요(설정은 즉시 반영되지 않고 다음 상호작용부터 반영됩니다).
- **`npm run build`가 실패해요**: `node --version`으로 22 이상인지 확인하세요.
- **Windows에서 빌드한 실행파일 실행 시 경고가 떠요**: 아직 디지털 서명이 없어 "알 수 없는 게시자" 경고가 뜰 수 있습니다. "추가 정보" → "실행"으로 진행하세요(정식 서명 배포 전까지는 정상입니다).
- **컨텍스트 퍼센트가 이상해요**: 세션 초반이나 `/compact` 직후에는 값이 비어있을 수 있습니다(정상 동작, Claude Code 공식 동작 방식).

## 자주 묻는 질문 (FAQ)

- **설치하면 자동으로 제 계정 정보를 가져가나요?** 아니요. 계정 관련 코드는 이 버전에 아예 포함되어 있지 않습니다.
- **인터넷으로 뭔가 전송되나요?** 아니요, 전부 로컬에서만 동작합니다.
- **나중에 계정 자동전환 기능이 생기면 자동으로 켜지나요?** 아니요. 여러분이 직접 동의해야만 켜지도록 설계되어 있습니다.
