# ClaudeTower (가제)

[한국어](./README.md) | [English](./README.en.md)

## 이게 뭔가요? (초보자용 한 줄 설명)

Claude Code를 쓸 때 화면 아래에 지금 상황(컨텍스트 사용량, 비용, 프로젝트 위치)을 보여주는 상태표시줄 도구입니다. 나중에는 여러 개인 계정을 자동으로 전환하는 기능도 추가될 예정이지만, 그 기능은 안전장치(사용자 동의 절차)가 완성되기 전까지는 절대 켜지지 않습니다.

## 지금 상태 (중요)

이 프로젝트는 아직 **Phase 1(첫 단계) 진행 중**입니다. 아직 일반 사용자가 바로 설치해서 쓸 수 있는 배포판은 없습니다 — 지금은 개발자가 소스 코드를 직접 받아 빌드/테스트하는 단계입니다.

- 완료: CLI 뼈대(`claudetower --version`/`--help`), 단일 실행파일(SEA) 빌드 파이프라인, Display/Account 모듈 코드 격리 구조 + 자동 검증
- 아직 없음: 실제 상태표시줄 위젯, `setup` 설정 마법사, 일반 사용자용 설치 스크립트

## 사전 준비물

- [Node.js](https://nodejs.org) 22 이상 (지금은 개발/빌드용으로만 필요 — 최종 배포판은 Node.js 설치 없이도 실행되도록 만들 예정입니다)
- Git

## 1. 저장소 받기

```bash
git clone <저장소 주소>
cd ClaudeTower
```

## 2. 필요한 도구 설치

```bash
npm install
```

개발에 필요한 도구(코드 검사기 ESLint, 빌드 도구 esbuild 등)를 받습니다. 인터넷 연결이 필요합니다.

## 3. 실행해보기

```bash
node bin/claudetower.js --help
node bin/claudetower.js --version
```

## 4. 테스트 돌려보기

```bash
npm test
```

코드 스타일 검사 + 모듈 경계(보안) 검사 + 테스트를 한 번에 전부 확인하려면:

```bash
npm run verify
```

## 5. 배포용 실행파일 빌드하기

```bash
npm run build
```

`dist/` 폴더 안에 여러분의 운영체제에 맞는 실행파일이 하나 생깁니다(예: Windows는 `claudetower-win-x64.exe`). 이 실행파일은 Node.js가 설치되어 있지 않은 컴퓨터에서도 그대로 실행됩니다.

## 자주 발생하는 문제

- **`npm install`이 안 돼요**: 인터넷 연결과 Node.js 설치 여부를 확인하세요(`node --version`으로 버전 확인).
- **`npm run build`가 실패해요**: Node.js 버전이 22 미만이면 실패할 수 있습니다.
- **Windows에서 빌드한 실행파일을 실행하면 경고가 떠요**: 아직 디지털 서명이 없어서 "알 수 없는 게시자" 경고가 뜰 수 있습니다. "추가 정보" → "실행" 순서로 진행하면 됩니다(정식 배포 전까지는 정상적인 현상입니다).

## 보안·구조 안내

- 이 프로그램은 두 부분으로 나뉘어 있습니다: **상태표시줄 부분**(항상 안전, 자격증명을 전혀 다루지 않음)과 **계정 자동전환 부분**(아직 코드 자체가 없음, 나중에 별도 동의 절차 후에만 추가·활성화).
- 두 부분은 코드 레벨에서 서로 접근할 수 없도록 강제되어 있으며, 이를 자동 검사(ESLint 규칙 + 별도 검증 스크립트 + 실행 테스트)로 매번 확인합니다(`npm run verify`).
- 자세한 설계 배경과 보안 요구사항은 [`.PRD/`](./.PRD/) 폴더의 문서를 참고하세요.
