## 1. 이 프로그램은 무엇인가요?

**ClaudeTower**(가제, 정식 이름은 아직 확정되지 않았습니다)는 **Claude Code**(AI에게 코딩을 시키는 프로그램)를 쓸 때 화면 맨 아래에 지금 상황을 한눈에 보여주는 작은 보조 프로그램입니다.

예를 들어 이런 식으로 화면 맨 아래 한 줄에 표시됩니다:

```
모델 Sonnet 5  📁 my-project  컨텍스트 ██░░░ 45%  💰 $1.50  5시간 ████░ 78%·1:41  7일 ███░░ 71%·일06:00
```

이 한 줄이 알려주는 정보:
- **모델 Sonnet 5** — 지금 어떤 AI 모델을 쓰고 있는지
- **📁 my-project** — 지금 작업 중인 폴더 이름
- **컨텍스트 ██░░░ 45%** — AI가 지금까지의 대화 내용을 얼마나 "기억"하고 있는지(꽉 차면 AI가 예전 대화를 잊어버립니다)
- **💰 $1.50** — 지금까지 이 대화에서 쓴 비용(달러)
- **5시간 ████░ 78%·1:41** — 5시간 단위 사용량 한도를 78% 썼고, 1시간 41분 뒤에 다시 채워진다는 뜻
- **7일 ███░░ 71%·일06:00** — 7일 단위 사용량 한도를 71% 썼고, 일요일 오전 6시에 다시 채워진다는 뜻

**가장 중요한 것부터 말씀드리면**: 이 프로그램은 **여러분의 Claude 계정 아이디·비밀번호·인증 정보를 전혀 다루지 않습니다.** 화면에 정보를 예쁘게 보여주기만 하는, 안전한 프로그램입니다.

## 2. 시작하기 전에 알아두어야 할 것

- 이 프로그램은 아직 **개발 초기 단계(Phase 1)**입니다. 지금 설명하는 "상태표시줄" 기능은 실제로 작동하지만, "여러 Claude 계정을 자동으로 바꿔가며 쓰는 기능"(계정 자동전환)은 **코드 자체가 아직 만들어지지 않았습니다.**
- 정식 이름이 아직 정해지지 않아 지금은 "ClaudeTower"라는 가제(임시 이름)를 쓰고 있습니다.
- 이 프로젝트의 GitHub 저장소는 **공개(public)**입니다. 다운로드 페이지를 보는 데 GitHub 로그인이 필요하지 않습니다.
- **무료이며, 개인 사용 목적입니다.** 상업적으로 판매하거나 회사에 유료로 납품하는 용도로는 만들어지지 않았습니다(자세한 내용은 17장 참고).

## 4. 다운로드 방법

1. 웹 브라우저를 엽니다(GitHub 로그인은 필요 없습니다).
2. 아래 주소로 들어갑니다.
   ```
   https://github.com/sodam-ai/ClaudeTower/releases/latest
   ```
3. 이 페이지는 "최신 버전 다운로드" 페이지입니다. 화면을 아래로 내리면 **"Assets"**(첨부 파일)라는 항목이 보입니다. 그 안에서 내 컴퓨터 운영체제에 맞는 파일을 클릭해서 받습니다.

   | 내 컴퓨터 | 받을 파일 이름 |
   |---|---|
   | Windows (대부분의 일반 PC) | `claudetower-win-x64.exe` |
   | macOS, Apple Silicon(M1/M2/M3 등) 맥북/아이맥 | `claudetower-macos-arm64` |
   | Linux, 64비트 | `claudetower-linux-x64` |

4. 다운로드가 끝나면 보통 "다운로드" 폴더(Windows는 `내 PC → 다운로드`)에 파일이 저장됩니다.

> ⚠️ **404 Not Found(페이지를 찾을 수 없음) 오류가 뜨면?** 주소를 정확히 입력했는지 다시 확인하세요(대소문자·오타 주의).

## 5. 빠른 시작 (설치)

### 5-1. 파일을 원하는 폴더로 옮기기

받은 파일을 아무 폴더에나 두셔도 됩니다. 예시로 `D:\Test_Dev\test1` 같은 폴더를 만들어서 옮겨두는 걸 권장합니다(찾기 쉬우니까요). 파일 이름을 바꾸셔도 됩니다.

### 5-2. 터미널(명령어를 입력하는 검은 창) 열기

"터미널"이란 글자로 명령을 입력해서 컴퓨터에 지시를 내리는 창입니다. 처음 보면 낯설 수 있지만, 아래 방법대로 하면 어렵지 않습니다.

**Windows**:
1. 파일이 있는 폴더를 파일 탐색기로 엽니다.
2. 창 위쪽의 **주소창**(폴더 경로가 적힌 부분)을 클릭합니다.
3. 글자를 전부 지우고 `cmd`라고 입력한 뒤 Enter 키를 누릅니다.
4. 검은 배경의 창이 뜹니다 — 이게 터미널입니다.

**macOS**:
1. `Finder`에서 파일이 있는 폴더를 엽니다.
2. 폴더 안 빈 공간에서 마우스 오른쪽 버튼(또는 두 손가락으로 트랙패드 클릭) → "폴더에서 새로운 터미널 열기"를 선택합니다.

### 5-3. 설치 마법사 실행하기

터미널에 아래처럼 입력하고 Enter를 누릅니다(파일 이름은 실제로 받으신 파일명으로 바꾸세요):

**Windows**:
```
claudetower-win-x64.exe setup
```

**macOS / Linux**:
```
./claudetower-macos-arm64 setup
```

- Windows에서 **"Windows가 PC를 보호했습니다"**라는 파란색 경고창이 뜰 수 있습니다. 이건 아직 이 프로그램이 정식 인증서를 받지 않은 초기 버전이라 그렇습니다(위험한 게 아닙니다). `추가 정보` 클릭 → `실행` 버튼을 누르면 계속 진행됩니다.
- 화면에 질문이 5개 뜹니다. 보여주고 싶은 항목이면 그냥 Enter(기본값은 "예"), 안 보이게 하고 싶으면 `n`을 입력한 뒤 Enter를 누릅니다.
  ```
  사용 모델 표시할까요? (Y/n):
  프로젝트 위치 표시할까요? (Y/n):
  컨텍스트 사용량 표시할까요? (Y/n):
  비용 표시할까요? (Y/n):
  사용률(5시간/7일) 표시할까요? (Y/n):
  ```
- 마지막에 "설정 완료"와 "안전한 위치로 복사했습니다"라는 문구가 뜨면 성공입니다.

### 5-4. Claude Code에서 확인하기

Claude Code를 새로 열거나(또는 새 대화를 시작하거나), 다음 대화를 진행하면 화면 맨 아래에 상태표시줄이 나타납니다. **컴퓨터를 재시작할 필요는 없습니다.**

## 6. 실행 방법

### 그냥 더블클릭하면 되나요?

파일을 더블클릭하면 검은 창이 잠깐 떴다가 바로 사라집니다 — **이건 고장이 아닙니다.** 이 프로그램은 "명령어"와 함께 실행해야 뭔가를 하는 프로그램이라, 아무 명령어 없이 그냥 실행하면 사용법만 보여주고 끝납니다.

**실제로 쓰려면 반드시 5장에서 설명한 것처럼 터미널을 열어서 명령어를 직접 입력해야 합니다.**

### 매번 터미널을 열어야 하나요?

아니요. **설치(`setup`)는 딱 한 번만 하면 됩니다.** 그 이후로는 Claude Code를 쓸 때마다 이 프로그램이 자동으로, 화면 뒤에서 조용히 실행되어 상태표시줄에 정보를 보여줍니다. 터미널을 다시 열어야 하는 경우는:
- 표시 항목을 바꾸고 싶을 때(`setup` 재실행)
- 설치가 잘 되어 있는지 확인하고 싶을 때(`status`)
- 프로그램 등록을 지우고 싶을 때(`uninstall`)

## 7. 사용 방법

### 표시 항목을 나중에 바꾸고 싶어요

`setup`을 다시 실행하면 됩니다. 5개 질문에 다시 답하면 그 즉시 반영됩니다. 예를 들어 "컨텍스트"와 "비용"은 그만 보고 싶다면, 그 두 질문에만 `n`을 입력하고 나머지는 Enter를 누르면 됩니다.

### 지금 설치가 잘 되어 있는지 확인하고 싶어요

```
claudetower-win-x64.exe status
```
아래처럼 나오면 정상입니다:
```
설치 상태: 설치됨 (claudetower 상태표시줄이 Claude Code에 등록되어 있습니다)
표시 중인 항목: 사용 모델, 프로젝트 위치, 컨텍스트 사용량, 비용, 사용률(5시간/7일)
```

### 더 이상 쓰고 싶지 않아요

```
claudetower-win-x64.exe uninstall
```
Claude Code에 등록된 상태표시줄 설정만 깔끔하게 지워집니다. 다른 Claude Code 설정(예: 여러분이 따로 설정해두신 것들)은 전혀 건드리지 않습니다.

### 원본 다운로드 파일을 지워도 되나요?

**네, `setup`을 한 번이라도 실행하셨다면 지우셔도 됩니다.** `setup`을 실행하는 순간 프로그램이 스스로 컴퓨터 안의 안전한 고정 위치(`~/.claudetower/bin/` 폴더)로 자기 자신을 복사해두기 때문에, 원래 받으신 파일은 그 뒤로 지우거나 옮기거나 이름을 바꾸셔도 전혀 문제가 없습니다.

## 8. 작동 방법 (원리)

쉬운 비유로 설명드리면:

Claude Code는 대화할 때마다 "지금 상황"(어떤 폴더에서 작업 중인지, 모델이 뭔지, 비용이 얼마인지 등)을 이 프로그램에게 살짝 알려줍니다. 이 프로그램은 그 정보를 받아서 "보기 좋게 정리한 한 줄"로 바꿔서 다시 Claude Code에게 돌려줍니다. Claude Code는 그걸 화면 맨 아래에 그대로 보여줍니다.

이 과정은 사람이 "안내판을 만들어주는 사람"을 옆에 앉혀둔 것과 비슷합니다 — 안내판 만드는 사람은 여러분이 어디 있는지(폴더), 지금 몇 시인지(사용률 재설정 시간) 등을 보고 표지판만 만들어줄 뿐, 여러분의 지갑이나 신분증(계정 정보)에는 손대지 않습니다.

이 프로그램은 **인터넷에 아무것도 전송하지 않습니다.** 오직 여러분의 컴퓨터 안에서만 정보가 오갑니다.

## 9. 명령어 전체 목록

| 명령어 | 하는 일 | 예시 |
|---|---|---|
| `claudetower --version` | 지금 설치된 버전 번호를 보여줍니다 | `0.1.7` |
| `claudetower --help` (또는 인자 없이 실행) | 사용법을 보여줍니다 | — |
| `claudetower setup` | 표시 항목을 고르고 Claude Code에 자동 등록합니다(설치/설정변경 겸용) | 5개 질문에 Y/n으로 답함 |
| `claudetower status` | 지금 설치 상태와 표시 중인 항목을 확인합니다 | "설치됨" / "설치 안 됨" / "고장 상태" 중 하나가 나옵니다 |
| `claudetower uninstall` | Claude Code에 등록된 상태표시줄 설정만 안전하게 제거합니다 | 다른 설정은 그대로 둠 |
| `claudetower statusline` | Claude Code가 내부적으로 자동 호출하는 명령입니다 | **직접 실행하실 일은 없습니다** |

> `accounts`, `config` 같은 계정 관련 명령어는 **아직 존재하지 않습니다**(Phase 2에서 추가될 예정, 지금은 코드 자체가 없습니다).

```
[2] 터미널 열기
      ↓
[3] "claudetower ... setup" 실행 → 5개 질문에 답변
      ↓
[4] 자동으로 안전한 위치에 복사됨 + Claude Code 설정에 등록됨
      ↓
[5] Claude Code에서 새 대화 시작 → 상태표시줄 확인
```

**평소 사용할 때**: 아무것도 안 하셔도 됩니다. Claude Code를 쓸 때마다 자동으로 화면 아래에 표시됩니다.

**표시 항목을 바꾸고 싶을 때**:
```
[1] 터미널 열기 (아무 폴더에서나 가능)
      ↓
[2] "claudetower ... setup" 다시 실행 → 원하는 항목만 Y, 나머지는 n
      ↓
[3] 다음 대화부터 바로 반영됨
```

**문제가 생겼을 때**:
```
[1] "claudetower ... status" 실행해서 상태 확인
      ↓
[2] "고장 상태"라고 나오면 → "claudetower ... setup" 다시 실행 (자동 복구)
      ↓
[3] 그래도 안 되면 → 15장 "문제 해결" 참고
```

## 13. 아키텍처

**비유로 설명하면**: 이 프로그램은 두 개의 완전히 분리된 방으로 설계되어 있습니다.

- **"상태표시줄 방"**: 지금 실제로 작동하는 부분입니다. 화면에 정보를 보여주기만 하는 방이라 항상 안전합니다.
- **"계정 자동전환 방"**: 아직 **지어지지도 않았습니다**(코드가 없습니다). 나중에 만들어지더라도, 상태표시줄 방과는 열쇠(권한)가 전혀 통하지 않는 완전히 별도의 방으로 설계될 예정입니다.

이렇게 나누는 이유는, 나중에 "계정 자동전환" 기능이 추가되더라도 그 기능에 문제가 생기는 것과 별개로 "상태표시줄" 기능은 항상 안전하게 계속 작동하도록 하기 위해서입니다. 실제로 이 분리가 잘 지켜지고 있는지는 프로그램을 만들 때마다 자동화된 검사 도구가 매번 확인합니다.

## 14. 파일 · 문서 위치

이 프로그램이 여러분 컴퓨터에 실제로 만들거나 사용하는 파일들의 위치입니다.

| 파일/폴더 | 위치(Windows 기준) | 무엇인가요? |
|---|---|---|
| 설치된 실행 파일 | `C:\Users\사용자이름\.claudetower\bin\claudetower.exe` | `setup` 실행 시 자동으로 복사되는, 실제로 작동하는 파일(진짜 "본체") |
| 표시 항목 설정 | `C:\Users\사용자이름\.claudetower\config.json` | 어떤 항목을 보여줄지 저장한 작은 파일 |
| Claude Code 전역 설정 | `C:\Users\사용자이름\.claude\settings.json` | Claude Code 자체의 설정 파일. 이 프로그램은 이 파일 안의 "statusLine" 부분만 사용하고 나머지(다른 설정들)는 절대 건드리지 않습니다 |
| 설정 백업 파일 | `C:\Users\사용자이름\.claude\settings.json.bak` | 설정을 바꾸기 직전의 원본이 자동으로 백업되는 파일 — 문제가 생기면 이 파일을 보고 되돌릴 수 있습니다 |

> macOS/Linux에서는 `C:\Users\사용자이름` 대신 `~`(홈 폴더, 보통 `/Users/사용자이름` 또는 `/home/사용자이름`)를 사용합니다.

**개발자용 설계 문서**: 이 프로그램이 왜 이렇게 만들어졌는지, 어떤 고민을 거쳤는지에 대한 상세한 기록은 저장소 안의 `.PRD/` 폴더에 전부 정리되어 있습니다(일반 사용자는 안 보셔도 됩니다).

## 15. 문제 해결 · 오류 대처 방법

| 증상 | 원인 / 해결 방법 |
|---|---|
| Releases 페이지에서 **404 오류**가 떠요 | 주소를 정확히 입력했는지 확인하세요. |
| 파란 경고창에서 **"실행" 버튼이 안 보여요** | "추가 정보"를 먼저 눌러야 그 아래에 "실행" 버튼이 나타납니다. |
| 파일을 **더블클릭했더니 창이 바로 꺼져요** | 고장이 아닙니다. 인자 없이 실행하면 도움말만 보여주고 끝납니다. 6장 "실행 방법"을 참고해 터미널에서 직접 명령어를 입력하세요. |
| `status`에서 **"등록은 되어 있으나 실행 파일을 찾을 수 없습니다(고장 상태)"**가 떠요 | 설치된 파일을 실수로 지우신 경우입니다. `setup`을 다시 실행하면 자동으로 복구됩니다. |
| Claude Code 화면에 **상태표시줄이 안 보여요** | 설정은 "다음 대화부터" 반영됩니다. Claude Code와 한 번 더 대화해보세요. |
| **컨텍스트 퍼센트가 이상하거나 비어있어요** | 대화 시작 직후나 `/compact`(대화 정리) 직후에는 일시적으로 비어있을 수 있습니다. 이건 Claude Code 자체의 정상적인 동작입니다. |
| 개발자용 — `npm run build`가 실패해요 | `node --version`으로 Node.js가 22 이상인지 확인하세요. |

**그래도 해결이 안 되면**: `claudetower status` 명령을 실행한 화면을 캡처(스크린샷)해서 문의해주시면 원인 파악이 훨씬 빨라집니다.

## 16. 자주 묻는 질문 (FAQ)

**Q. 설치하면 제 Claude 계정 정보를 가져가나요?**
A. 아니요. 계정 관련 코드는 이 버전(v0.1.7)에 아예 포함되어 있지 않습니다. 이 프로그램은 여러분의 아이디·비밀번호·인증 토큰을 볼 수도, 저장할 수도 없는 구조입니다.

**Q. 인터넷으로 뭔가 전송되나요?**
A. 아니요. 전부 여러분 컴퓨터 안에서만 동작합니다.

**Q. 나중에 "계정 자동전환" 기능이 생기면 자동으로 켜지나요?**
A. 아니요. 그 기능은 여러분이 직접 명시적으로 동의해야만 켤 수 있도록 설계될 예정이며, 켜지 않아도 상태표시줄 기능은 계속 정상 작동합니다.

**Q. 돈이 드나요?**
A. 아니요, 무료입니다. 다만 이 프로그램 자체와, 이 프로그램이 화면에 표시해주는 "비용($)" 정보는 서로 다릅니다 — 그 비용은 여러분이 Claude Code(AI)를 사용하면서 발생하는 별도의 비용이며, 이 프로그램과는 무관합니다.

**Q. 원본 다운로드 파일을 지워도 정말 괜찮나요?**
A. 네, `setup`을 한 번이라도 실행하셨다면 괜찮습니다. 7장 참고.

**Q. 왜 프로그램 이름이 "가제"라고 되어 있나요?**
A. 정식 이름이 아직 확정되지 않았습니다(상표 저촉 여부 등 법률 검토가 끝나지 않은 상태입니다). 이름이 정해지면 안내드릴 예정입니다.

**Q. 이 프로그램을 회사에서 써도 되나요?**
A. 개인 사용을 전제로 만들어졌으며, 상업적 판매나 회사 납품 목적으로는 만들어지지 않았습니다. 자세한 내용은 아래 17장을 반드시 읽어주세요.
